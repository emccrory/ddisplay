gov.fnal.ppd.dd.changer.ChannelClassification Class Reference

Detailed Description

The way the channels are organized/sorted for the user.

This corresponds to the field ChannelTabSort.Type in the database. (I really need to unify these names!)

TODO - Currently, there is a one-to-one relationship between Channels and ChannelClassification's. This might not be the right choice. In other words, why can't a Channel be associated with two categories?

FIXME - There seems to be some confusion. There is the ChannelTabSort.Type field, and then this class is used to distinguish between regular, web-based content and content that comes directly from an image.

April 2018: Removed the pre-defined categories. This is held exclusively in the database
