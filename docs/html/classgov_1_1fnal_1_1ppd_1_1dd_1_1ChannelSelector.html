Detailed Description

This is the class that is used to create the GUI for the so-called Dynamic Displays Channel Selector.

Allows the user to select the Channel that is being displayed on each Display in the Dynamic Displays system

The configuration of the GUI is determined by a series of system parameters, which are held in gov.fnal.ppd.dd.GlobalVariables, in the DB and in the PropertiesFile.

TODO - Refactor this complicated class. There are several aspects of the channel selector that are mixed together in this class:

  • Building the GUI (Commented with STARTUP)
  • maintaining the GUI (e.g., responding to user input; RUN_GUI),
  • dealing with the logic of the channel selector (LOGIC),
  • presenting the user with various bits of "eye candy" (CANDY).

A reasonable goal would be to split this into more testable pieces (e.g., using the model-view-controller pattern). More simply, It seems like we should be able to factor out the building of the GUI (at least).
