Detailed Description

The bare-bones implementation of a server that will distribute messages to clients that are subscribed to topics.

This version of the server can only exist within the JVM where everything is running. The implementation in the Dynamic Displays system would need to have this class be in a separate JVM, on a different node on the Internet.

The way I would probably implement this is the same way that Displays were implemented - with a Facade within the JVM that the client connects to, and them that Facade will have the logic to contact the server on the Internet.

https://riptutorial.com/design-patterns/example/6498/publish-subscribe-in-java (Retrieved AWpt 2021)
